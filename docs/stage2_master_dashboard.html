MAP 是什么?给定 EEG 观测,4 个生理参数最可能的取值。 每个参数 2 根 bar:■ 蓝色 = 用 7 个 EEG 特征推出来,■ 橙色 = 用 5 个 EEG 特征推出来。

4 个参数生理含义:

  • g_h:超极化激活电流(神经元慢节律驱动)
  • g_LK:K+ 漏电流(神经元静息状态)
  • c_ctx2th:皮层→丘脑突触连接强度(我们最关心的参数)
  • b:适应电流(神经元疲劳/适应)

怎么看:两根 bar 越接近 → 这个参数稳定可识别(不管用几个特征都推出同一个值)。 看 c_ctx2th:蓝 0.069 → 橙 0.206,差 +198% → 不稳定 = 不可识别, 这就是 Tab 3 详细诊断的对象。

SBC 是什么?给算法做的"自检":从 prior 随机生成 200 组真值参数, 每组跑一次仿真,然后让算法反推后验,看后验包不包对真值。 如果算法过度自信(后验太窄)或不够自信(后验太宽),都会被这个检验抓到。

怎么看:每根绿色 bar 长度 = KS p-value。 红色虚线在 p=0.05 阈值。bar 越过红线右边 = PASS (算法对这个参数校准合格)。

大字 "4/4 PASS" 含义:4 个参数全部通过 → NSF 神经网络在 5 维 likelihood 上 学到了正确的算法映射。这意味着 Tab 1 看到的 c_ctx2th 4 模态不是算法 bug, 是数据本身就这样,这一点对 paper 论证非常关键。

⚠️ 注意 b 的 p=0.0850 离 0.05 阈值只剩 0.035 余量,虽然 PASS 但贴边, Sprint 2 可考虑增加 SBC 样本数复测。

PPC 是什么?从训练好的后验抽 100 组参数,跑 100 次仿真, 看真实 EEG 观测 x_obs 落在这 100 个仿真值的哪个百分位。 理想情况是落在中间(后验仿真分布包住 x_obs)。

怎么看:每根 bar 长度 = 百分位(0% 到 100%)。 判据是落在 [5%, 95%] 区间 = PASS。

  • ■ 绿色 PASS:仿真器能产生包含 x_obs 的输出分布
  • ■ 红色 FAIL:x_obs 落在仿真分布之外(0% 或 100%)
  • ■ 灰色 design:shape_r=1.0 是我们硬编码的设计哨兵,100% FAIL 是预期,豁免

3/5 PASS 解读:T4_q=100% 是新发现的偏移项 — 后验仿真的 SO 峰 Q 因子全都低于真实 EEG。这不影响 c_ctx2th 主结论,但 Sprint 2 需要单独跟进。

Pareto Seed 是什么?Stage 1(进化算法 DE)找到的 3 组有代表性 的最优参数,各代表不同"权衡":

  • Seed A — PAC-dominant:SO-spindle 耦合强度最高(MI=0.137)
  • Seed B — 综合 fitness 最高(score=0.674,DE 排名第一)
  • Seed C — shape-dominant:谱形匹配最好(shape_r=0.690)

log_prob 怎么读?后验在该点的对数概率密度。 值越大(=越靠右)→ 这个点越靠近后验高密度区,SBI 越"认可"它。 值越负 → 越被后验排除。

怎么看:每个 Seed 有 2 根 bar: ■ 蓝色 = 7 维后验下, ■ 橙色 = 5 维后验下。

关键发现:排序从 7 维 B>C>A → 5 维 B>A>C, Seed A 和 Seed C 的相对位置反转。原因:5 维删除了 MI(PAC 强度指标), 所以 PAC-dominant 的 A 和 shape-dominant 的 C 不再被区分。 这是 likelihood 维度选择直接影响后验排序的硬证据。

"边际后验"是什么?SBI 训出来的是 4 维联合后验(4 个参数的联合概率分布)。 把其他 3 个参数(g_h、g_LK、b)"积分掉"(数学上叫 marginalize),只看 c_ctx2th 这一维上的分布形状,就是边际后验。可以理解成"不管其他参数怎么变, c_ctx2th 单独看下来在哪些值上密度高"。

怎么看大图(左上):

  • 每个"山头"= 一个 mode(后验偏好的取值);山越高 → 后验密度越大
  • 上半 May 7(蓝色,7 维):3 个 mode,主峰在低端 0.075, 小峰在 0.21。整体偏向 prior 下半区。
  • 下半 May 23(橙色,5 维):4 个 mode 跨越整个 prior (0.052 / 0.107 / 0.155 / 0.207),主峰跳到上端 0.207。
  • 底部灰条:prior [0.05, 0.22] 全宽,提供尺度参考

关键观察:不同的 likelihood 维度(7 维 vs 5 维)给出 完全不同的模态结构——真实可识别的参数不应有这种敏感性。 打个比方:如果你用两种不同的尺(7 个刻度 vs 5 个刻度)量同一个物体, 长度应该差不多;但 c_ctx2th 给的结果是"完全不同的形状"——说明这个"物体" 其实根本不能被尺测到,你看到的只是尺刻度本身的伪影。

怎么看底部 95% CI 横条:

  • "95% CI" 是什么:后验质量 95% 集中在哪个区间。 理想情况下,可识别参数的 CI 会非常窄(只占 prior 的一小段)
  • 这里 CI = [0.052, 0.214],跨度占 prior 全宽 95%
  • 意思是:EEG 数据几乎没缩小 c_ctx2th 的不确定性—— 后验和 prior 在这个维度上几乎一样宽,等于"没学到东西"
看图指南(2/2):右栏 3 个 Pairplot mini panel

"pairplot"是什么?把后验在两个参数维度上的联合分布画出来, 看两者之间有没有结构性关系。可识别参数的 pairplot 应该呈现"集中的椭圆" 或"斜对角的山脊"(ridge);横条或竖条意味着其中一个参数 被另一个参数"决定不了"——即两者独立。

3 个 panel 的形态:

  • c_ctx2th × g_h(横条):水平方向的细带跨越全 prior。 意思是不管 g_h 取什么值,c_ctx2th 几乎可以是任何值 → g_h 不约束 c_ctx2th
  • c_ctx2th × g_LK(横条):同上,g_LK 也不约束 c_ctx2th
  • c_ctx2th × b(竖条):4 个垂直亮带堆叠,但 b 维度极窄(50 附近)。 意思是 b 紧紧锁定,c_ctx2th 的 4 个 mode 共享同一个 b 值 → b 也不 解释 c_ctx2th 的多模态来源

"无 ridge"的含义:如果两参数有耦合,你会看到斜对角的"山脊带" (比如 c_ctx2th 大时 b 也大)。这里 3 个 panel 都没有 ridge → 没有任何其他参数在"解释"c_ctx2th 的变异 → c_ctx2th 的所有不确定性都 来自 likelihood 信息不足,不是被其他参数耦合掉了。

三条证据合起来(marginal 4 模态 + pairplot 无 ridge + CI 占 prior 95%): c_ctx2th 不是"测出来一个 95% CI 偏宽但峰值清晰"的参数, 而是"模态结构本身随观测维度变化"的结构性不可识别参数。 这就是 Stage 2 论文的核心发现。

这一页讲什么?Sprint 1 Phase 2 的"补救尝试"为什么失败,为什么我们最终 选择走"路径 W"(接受 5 维 baseline 作为最终结果)。整个 Phase 2 持续了 May 23 一整天,顶部 5 步流程框就是这一天的真实时间顺序。

顶部 5 步流程框(左→右):

  • 阶段 A 合成 5 case PASS(白框):在合成 ground-truth 信号上跑算法, 10/10 通过判据 → 算法本身没 bug,可以进真实数据测试
  • 阶段 B step 5 SC4001 MI=0.00019(红框): 真实 EEG 上测出来的 MI 比预期低 100×(预期 [0.02, 0.05])→ 触发 algorithm-suspect FAIL,暂停
  • Quick verification(白框):猜想"epoch 拼接是主因", 测试 480s 零边界连续段 → MI 仍 = 0.0001 → 排除拼接根因
  • Surrogate test(红框): 换更严格的统计检验,看 MI 是不是高于"零信号"水平 → 仍然 null(z=0.14)→ 排除"信号弱但显著"的可能
  • 走路径 W(绿框):MI 维度物理上不可用, 放弃增加 likelihood 维度,5 维 baseline 作为最终结果

中部 5 个 z-score 卡片:

"z-score" 是什么?衡量观测到的 MI 比"零信号假设"高多少个标准差。 可以理解成"PAC 信号比噪声强多少倍"的标准化指标。 判据:|z| > 2 ≈ 显著(p < 0.05),说明 PAC 真实存在; |z| < 2 = 在 null 范围内,统计上不能区分于无信号。

怎么看每张卡:大数字 = z 值,左边色条标颜色:

  • ■ 橙色 warn(Mode X +21.40): 看起来"显著",但实际是 boundary artifact — 4260s 信号是 142 个 非连续 epoch 拼接而成,141 个边界让 null σ 异常窄,人为推高 z
  • ■ 灰色 null(Mode Y +0.14 和白噪 -1.70): Mode Y 是 480s 连续段(0 边界),z=+0.14 与白噪的 -1.70 在 ±2 内统计不可区分 → SC4001 真实 PAC ≈ 白噪
  • ■ 绿色 good(Simulator +42.51 和 +68.72): 仿真器侧 PAC 真实可测且强烈;single-channel (r_ctx × r_ctx) z=+68.72 比 cross-channel (r_ctx × r_thal) z=+42.51 更高,因为 r_ctx 自身包含完整 皮层-丘脑共耦合电路输出

"白噪 paranoia" 是什么?专门跑一遍纯白噪信号作为 algorithm self-check — 如果算法本身有 bug,白噪也会给出高 z;实际白噪 z=-1.70(在 ±2 内)→ 算法自身没问题,Mode Y 的 z=+0.14 是真实 null,不是算法故障。

关键对比:SC4001 EEG 端 z=+0.14(null) vs Simulator 端 z=+42 (强显著),两端 z 量级差 ~300 倍。意思是 MI 这一维"对仿真器而言可观察、 对 EEG 数据而言不可测"——把它放进 7 维 SBI 会把 NSF 推向"破坏仿真器 PAC" 的参数空间,反而错。
